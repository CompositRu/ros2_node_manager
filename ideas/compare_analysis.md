## Tram Monitoring System — Описание проекта

**Tram Monitoring System** — веб-интерфейс для мониторинга и управления ROS2 нодами в Docker контейнерах. Ориентирован на операторов и разработчиков автономных систем (трамваи, роботы) для быстрой диагностики и управления большим количеством нод (~500+).

### Архитектура
```
Browser → FastAPI Backend → docker exec / SSH → ROS2 CLI
```

---

## Возможности

### ✅ Реализовано (MVP v0.1)

| Функция | Описание |
|---------|----------|
| **Дерево нод** | Иерархическое отображение по namespace с подсчётом |
| **Счётчики** | Total / Active / Inactive в реальном времени |
| **История нод** | Inactive ноды сохраняются, видно что было раньше |
| **Определение типа** | Автоматическое: Lifecycle / Regular / Unknown |
| **Детали ноды** | Parameters, Subscribers, Publishers, Services |
| **Shutdown** | Lifecycle ноды через `ros2 lifecycle set` |
| **Kill** | Regular ноды через `kill` (с подтверждением) |
| **Логи в реальном времени** | Стрим из /rosout с фильтрацией по ноде |
| **Локальный Docker** | Подключение к локальному контейнеру |
| **SSH подключение** | Удалённые серверы через SSH |
| **Кэширование** | Быстрое переключение между нодами |
| **WebSocket** | Real-time обновления статусов и логов |

## Выводы

### Tram Monitoring System лучше подходит для:
- **Управления нодами** (lifecycle, kill, групповые операции)
- **Мониторинга большого количества нод** (~500+)
- **Быстрой диагностики** без установки доп. пакетов
- **Работы через SSH** с удалёнными системами
- **Отслеживания истории** (какие ноды были/исчезли)

### Foxglove лучше подходит для:
- **Визуализации данных** (3D, графики, изображения)
- **Отладки алгоритмов** (просмотр топиков в реальном времени)
- **Работы с rosbag** (запись, воспроизведение)
- **Кастомизации** (extensions, layouts)

### Возможная интеграция
Вместо "или/или" можно использовать **вместе**:
- Tram Monitoring System — для управления и мониторинга нод
- Foxglove — для визуализации данных

Или реализовать недостающий функционал как **Foxglove Extension** (custom panel), но это потребует:
1. Установки foxglove_bridge в контейнер
2. Написания extension на TypeScript
3. Доступа к ROS2 CLI изнутри Foxglove (сложно)
